생성자

1. 인스턴스가 생성될 때 호출되는 '인스턴스 초기화 메서드'
2. 메서드처럼 클래스 내에 선언됨, 생성자의 이름 = 클래스의 이름 but 리턴값 없음.

3. 생성자간의 호출에는 this()를 사용해야함. (this와 다름)

4. 생성자를 포함한 모든 인스턴스메서드에는 자기참조변수 this가 지역변수로 숨겨진채로 존재함.


변수의 초기화

1. 명시적 초기화

변수를 선언과 동시에 초기화 하는 것.
ex) int door = 4; // 기본형
    Engine e = new Engine(); // 참조형

명시적 초기화가 간단하고 명료하긴 하지만, 보다 복잡한 초기화 작업이 필요할 경우
생성자 또는 초기화 블럭을 사용해야 함.

2. 생성자

3. 초기화 블럭

클래스 초기화 블럭과 인스턴스 초기화 블럭이 있음.
클래스 초기화 블럭은 클래스가 메모리에 처음 로딩될 때 한번만 수행됨.
인스턴스 초기화 블럭은 생성자와 같이 인스턴스를 생성할 때마다 수행됨.

수행 순서: 인스턴스 초기화 블럭 > 생성자

멤버변수의 초기화 시기와 순서

클래스변수 시점: 클래스가 처음 로딩될 때 단 한번.
인스턴스변수 시점: 인스턴스가 생성될 때마다, 각 인스턴스별로.

클래스변수 초기화 순서: 기본값 -> 명시적초기화 -> 클래스 초기화 블럭
인스턴스변수의 초기화 순서: 기본값 -> 명시적초기화 -> 인스턴스 초기화 블럭 -> 생성자
