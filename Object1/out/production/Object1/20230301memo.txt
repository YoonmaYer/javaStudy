변수와 메서드

변수는 선언위치에 따라 종류가 다름

멤버변수 중 static = 클래스변수, static != 인스턴스 변수, 이들이 아닌것은 지역변수

1. 인스턴스변수

독립적인 저장공간, 서로 다른 값 가질 수 있음
클래스 영역 안에 생성

2. 클래스변수
인스턴스변수 앞에 static을 붙여 선언
인스턴스를 생성하지 않고도 언제라도 바로 사용할 수 있음
전역변수랑 비슷함, 그러나 자바에서는 전역변수를 지원하지 않으므로 다른 개념(캡슐화, 은닉성)
public을 붙이면 전역변수의 성격을 가질 수 있음

3. 지역변수
메서드 내 선언, 사용 가능(local)
메서드가 종료되면 소멸됨

4. 메서드

함수와의 유사성
높은 재사용성
중복된 코드의 제거
프로그램의 구조화(main 메서드 안에 다른 메서드를 호출하는 식으로 구조를 단순하게 관리)


JVM의 메모리 구조

1. 메서드 영역

클래스에 대한 정보 저장

2. 힙

인스턴스변수가 생성되는 공간

3. 호출스택

메서드의 작업에 필요한 메모리 공간 제공, 지역변수들과 연산의 중간결과를 저장하는데 사용됨
작업 종료시 할당된 메모리공간은 소멸(지역변수의 특징)

오버로딩

한 클래스 내에 같은 이름의 메서드를 여러 개 정의하는 것

조건1. 메서드 이름이 같아야함
조건2. 매개변수의 개수 또는 타입이 달라야 함.
